# Psycho Portrait

**Автор проекта:** [Saborrr](https://github.com/Saborrr)

**Репозиторий:** [Saborrr/psycho-portrait](https://github.com/Saborrr/psycho-portrait)

**Лицензия:** [MIT](LICENSE)

Защищенное веб-приложение для преобразования PPTX/PDF с результатами психологического тестирования в структурированную рабочую характеристику и книгу Excel.

Приложение извлекает таблицы и исходные данные диаграмм напрямую из PPTX, проверяет достоверность профиля, обезличивает контекст для LLM, требует отчет установленной структуры и повторно валидирует результат.

> Это инструмент поддержки психолога, а не диагностическая система. Черновик нельзя использовать как единственное основание приема, увольнения, повышения или признания профессиональной непригодности.

## Возможности

- Структурное чтение таблиц и диаграмм вместо зависимости только от регулярных выражений.
- Поддержка реального формата приложенных карточек: 15 показателей СМИЛ, 6 диаграмм, интеллект, активность, эмпатия, отношение к дискомфорту и безопасности.
- Единый отчет из пяти частей: эмоциональная природа мотивации, стиль управления, стиль коммуникации, факторы риска и рекомендации.
- Не менее 100 слов в каждом из первых четырех разделов и не менее 10 практических рекомендаций. Ответ LLM проверяется, при ошибке выполняется одна корректирующая генерация.
- Пакетная загрузка документов и экспорт `.xlsx` с фотографиями сотрудников.
- Учет методики из лекции «Психологическое тестирование кандидатов», актуализация 30.01.2025.
- Пользовательское правило пятой шкалы имеет приоритет: направление одинаково для мужчин и женщин. Высокое значение трактуется в сторону феминизированности, низкое в сторону мускулинности. Высокая социальная интроверсия означает интровертированность.
- Адаптивная веб-панель для компьютера и телефона. При размещении по HTTPS ее можно установить на Android как PWA.

## Как лучше использовать

### 1. Локально на компьютере

Подходит для разработки и проверки. Сервер и браузер работают на одном компьютере. С телефона такой режим по умолчанию недоступен, поскольку Docker публикует порт только на `127.0.0.1`.

### 2. На сервере через веб-панель, рекомендуется

Приложение уже содержит веб-панель, поэтому отдельный frontend-сайт не нужен. FastAPI одновременно отдает интерфейс и защищенный API. Рекомендуемая схема:

```text
Компьютер или Android -> HTTPS/VPN -> Psycho Portrait -> обезличенный запрос -> LLM API
```

Размещайте сервис на собственном сервере или VPS. Оставьте контейнер привязанным к `127.0.0.1:8000`, а внешний доступ организуйте через reverse proxy с HTTPS либо через корпоративный VPN. Не публикуйте порт `8000` напрямую в интернет.

### 3. На Android

Отдельный APK не требуется. После размещения веб-панели по HTTPS:

1. Откройте адрес сервиса в Chrome на Android.
2. Выберите меню браузера -> «Установить приложение» или «Добавить на главный экран».
3. Запускайте Psycho Portrait с иконки как обычное приложение.
4. Введите `APP_API_KEY`, подтвердите полномочия и выберите PPTX/PDF через системное приложение «Файлы».

На телефоне хранится только оболочка интерфейса и API-ключ текущей вкладки. Загруженные файлы, ответы API и характеристики service worker не кэширует. Для обработки все равно нужны доступ к серверу и интернет/локальный LLM endpoint. Нативный APK имеет смысл только при необходимости публикации в Google Play, биометрии или корпоративного MDM.

## Как работает пакетная загрузка

1. В панели можно выбрать или перетащить несколько `.pptx`/`.pdf` одновременно.
2. Браузер показывает список выбранных файлов, но не читает их содержимое.
3. Кнопка «Сформировать Excel» отправляет файлы одним запросом в `/api/batch/generate-xlsx`.
4. Сервер последовательно проверяет контейнер, извлекает показатели и фото, обезличивает профиль, формирует и валидирует характеристику каждого сотрудника.
5. После успешной обработки всех файлов сервер создает одну книгу Excel в памяти: одна строка на сотрудника, пять разделов отчета, минимум 10 рекомендаций и фотография.
6. Готовый файл `psychological_characteristics.xlsx` автоматически скачивается в браузере.

По умолчанию разрешено до 25 файлов (`MAX_BATCH_FILES=25`). Для первых запусков разумно обрабатывать по 5-10 файлов: генерация выполняется последовательно, чтобы не превысить лимиты LLM-провайдера. Если один документ или отчет не проходит проверку, книга не выдается, а пользователь получает код ошибки. Реальные документы никогда не добавляются в Git.

## Защита персональных данных

- API закрыт `APP_API_KEY`. Без ключа приложение отказывает в обработке, кроме явно разрешенного локального режима разработки.
- ФИО, точная должность, имя файла, адрес, семейное положение, образование, история работы и исходный текст не отправляются LLM. Передаются только признак управленческой роли и обезличенные числовые результаты.
- История выключена по умолчанию (`STORE_HISTORY=false`).
- При включении истории обязателен Fernet-ключ. Профиль, отчет, ФИО и имя файла хранятся только в зашифрованном блоке; дедупликация выполняется по HMAC, а не по открытому SHA-256.
- Исходный текст не сохраняется, пока отдельно не включен `STORE_RAW_TEXT=true`.
- Срок хранения задается через `DATA_RETENTION_DAYS`, просроченные записи удаляются.
- API-ответы имеют `Cache-Control: no-store`; подробные ошибки и персональные данные не записываются в журнал.
- Ограничиваются размер, число файлов и слайдов, распакованный размер PPTX и коэффициент сжатия ZIP.
- В Docker процесс работает не от root, без Linux capabilities, с read-only файловой системой и отдельным томом данных.

Псевдонимизированные психометрические показатели все равно могут быть чувствительными данными. Для реальной эксплуатации проведите оценку рисков, оформите основание обработки и сроки удаления, ограничьте круг пользователей, настройте резервные копии с шифрованием и проверьте условия выбранного LLM-провайдера. Самый консервативный вариант: локальная модель внутри контролируемого контура. Подробнее: [docs/privacy.md](docs/privacy.md).

## Быстрый локальный запуск

```bash
git clone https://github.com/Saborrr/psycho-portrait.git
cd psycho-portrait
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Сгенерируйте ключ доступа к панели:

```bash
python -c "import secrets; print(secrets.token_urlsafe(48))"
```

Откройте `.env` и заполните как минимум `LLM_PROVIDER`, `LLM_API_KEY` и `APP_API_KEY`, затем:

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Откройте `http://127.0.0.1:8000`, введите `APP_API_KEY`, подтвердите полномочия на обработку и выберите один или несколько файлов.

На Windows проще установить Docker Desktop и использовать команду `docker compose up --build -d`. Для запуска через Python активируйте окружение командой `.venv\Scripts\Activate.ps1` в PowerShell вместо `source .venv/bin/activate`.

### Docker

```bash
docker compose up --build -d
```

Сервис публикуется только на `127.0.0.1:8000`. Для сетевого доступа используйте доверенный reverse proxy с TLS, аутентификацией и журналом доступа без содержимого документов.

## Запуск на сервере

Минимальный вариант для Ubuntu-сервера с установленными Docker Engine и Compose:

```bash
git clone https://github.com/Saborrr/psycho-portrait.git
cd psycho-portrait
cp .env.example .env
nano .env
docker compose up --build -d
docker compose ps
curl http://127.0.0.1:8000/health
```

После этого подключите домен к reverse proxy и направьте его на `http://127.0.0.1:8000`. HTTPS обязателен для передачи персональных данных и установки PWA. Секреты храните только в `.env` или серверном менеджере секретов, не в GitHub Actions, frontend-коде и не в APK.

Для небольшой личной установки достаточно `APP_API_KEY`. Для нескольких сотрудников организации перед публичным запуском добавьте SSO/OIDC, индивидуальные учетные записи, роли и аудит действий: общий API-ключ не позволяет определить, кто именно загрузил или удалил данные.

## Ключевые настройки

```env
LLM_PROVIDER=qwen
LLM_API_KEY=...
APP_API_KEY=длинная-случайная-строка

STORE_HISTORY=false
STORE_RAW_TEXT=false
DATA_RETENTION_DAYS=30
DATA_ENCRYPTION_KEY=

MAX_UPLOAD_MB=25
MAX_UNPACKED_MB=150
MAX_SLIDES=50
MAX_BATCH_FILES=25
```

Ключ шифрования для истории:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

Ключ нельзя коммитить, печатать в логах или хранить рядом с резервной копией базы. Потеря ключа делает историю невосстановимой.

## API

Все маршруты `/api/*` требуют заголовок `X-API-Key`.

| Метод | Маршрут | Результат |
|---|---|---|
| `POST` | `/api/parse` | Проверенный структурированный профиль без исходного текста |
| `POST` | `/api/generate` | Один структурированный отчет и Markdown |
| `POST` | `/api/batch/generate-xlsx` | Пакетная книга Excel |
| `GET` | `/api/settings` | Режим хранения и срок удаления |
| `GET` | `/api/sessions` | Зашифрованная история, если она включена |
| `DELETE` | `/api/sessions/{id}` | Удаление записи |

Пример:

```bash
curl -H "X-API-Key: $APP_API_KEY" \
  -F "file=@samples/sample_efko_full.pptx" \
  http://127.0.0.1:8000/api/generate
```

## Методологические ограничения

Пороговые диапазоны корпоративных методик перенесены в `app/methodology.py`. Достоверность СМИЛ проверяется до генерации. При значениях L/F/K выше 80 профиль считается ненадежным, диапазон 70-80 отмечается как сомнительный. Интерпретация строится по сочетаниям шкал и формулируется через вероятные рабочие проявления без технических названий и клинических терминов.

Корпоративные методики не являются общепопуляционными нормами. Поэтому приложение не выдает фиктивные перцентили и не маскирует внутренние пороги под стандартизованные нормативы.
