# 📋 Шаблон PPTX для Psycho Portrait

## Структура слайдов

### Слайд 1 — Шапка (обязательно)

```
ФИО: Синтетический Профиль
Должность: Мастер цеха №3
Подразделение: Цех молочной продукции
Возраст: 35
Пол: мужской
Стаж: 8
Образование: высшее
```

### Слайд 2 — 16PF Кеттелла (опционально)

Любой формат — парсер поймёт. Главное: код фактора (A, B, C, …) и число 1-10.

Пример 1 (одна строка):
```
16PF Кеттелла (стэны):
A: 7  B: 5  C: 6  E: 8  F: 5  G: 9  H: 6  I: 5
L: 4  M: 5  N: 6  O: 3  Q1: 5  Q2: 7  Q3: 7  Q4: 4
```

Пример 2 (таблица):
```
| Фактор | Стэн | Описание              |
|--------|------|-----------------------|
| A      | 7    | Замкнутость-общительность |
| B      | 5    | Интеллект            |
| ...    | ...  | ...                  |
```

## Советы

1. **Не смешивай разные методики на одном слайде** — парсеру легче, если каждая методика отдельно.
2. **Коды факторов — латиницей** (A, B, C, E, F, G, H, I, L, M, N, O, Q1, Q2, Q3, Q4) — парсер сначала ищет их.
3. **Баллы — числа 1-10** (стэны), 0-100 (T-баллы / проценты), 50±10 (MMPI T-баллы).
4. **Словесные описания** (например, "высокая тревожность") не заменяют числовой результат.
5. **Любой порядок слайдов** — парсер сам найдёт методики по ключевым словам.
6. **Если методика не распознана** — посмотри `notes` в ответе API и подправь формат.

## Что делать, если парсер ничего не нашёл

1. Вызови защищенный `POST /api/parse` и проверь `notes` и распознанные поля.
2. Сравни документ с шаблоном: значение должно быть числом, а диаграмма должна содержать встроенные данные Office.
3. Создавая issue, используй только полностью синтетический пример без ФИО, фотографии, адреса и результатов реального человека.
